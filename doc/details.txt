Общее описание.
Итак, ты - хакер, главный герой этой игры. Однажды, Тебе приходит письмо от незнакомца - некого дока, который выступает
в роли нанимателя на какую-то "грязную" работу. В письме, будет прикрепрел файл непонятного формата. Именно с этого всё
и начинается...

Уровень 1.
Ты получаешь непонятный файл, содержимое которого нужно извлечь. На самом деле это - 7z архив, пароль к которому нужно
подобрать. Автор рекомендует написать небольшую программу взломщик, которая будет перебирать пароли последовательно.
Данный алгоритм известен, как BruteForce. Перебор всех паролей должен занять около суток (от 6 до 18 часов в зависимости
от мощности компьютера). В качестве подсказки, можно указать на первые два байта архива. Они сигнализируют о формате
файла - '7z'. Результат работы нужно выслать доку на email.

Уроверь 2.
После распаковки 7z архива, ты получаешь другой файл - data.bin. Это зашифрованный специальным скриптом файл, интерпретатор
к которому нужно написать. В письме к этому уровню, док вышлет тебе скрипт, который его ребята утащили из переписки
чиновника и сотрудника NASA. Для упрощения задачи к уровню прилагаются тесты, которые должен пройти твой интерпретатор.
Разумеется, ты можешь их не использовать, если хочешь :). Но они значительно помогают, ведь одна оплошность и файл будет
расшифрован неверно... В файле answers.js находится json с ответами, которые ты должен получить запуская тесты. Например,
запустив тесты для команды set, нужно получить такие значения переменных:
set: {v1: 0, v2: 'str', v4: 12, v5: 'qwerty', n1: 2}
v1 - 0
v2 - 'str'
и так далее. Все остальные файлы в формате: command-errX.alan должны вызвать ошибку работы интепретатора.
После успешной расшифровки, ты должен получить файл term.html. Его уже можно запусть в браузере Chrome и только в нем.

Уроверь 3.
Для прохождения этого уровня тебе понадобится интернет.
Запускаем term.html в браузере - это программа удаленного терминала. А-ля UNIX консоль. Твоя задача подключиться к
спутнику с помощью этой консоли. Подключение осуществляется с помощью специальной команды - connect user password. Имя
пользователя и пароль - это то, что тебе нужно будет раздобыть. По умолчанию ты - гость (guest). По этому, большинство
файлов в этой системе для тебя недоступно. По всей системе (в каждой папке) раскиданы файлы с именем perm. Это файлы
прав пользователей. Они определяют какие файлы может просматривать, удалять и модифицировать текущий пользователь.
Все эти файлы не доступны для просмотра гостю. По-этому, тебе нужно найти такой, который ты сможешь прочесть. Один из
таких файлов находится в папке home. С помощью команды read perm находять в папке home ты сможешь вывести в консоль
содержимое файла perm и тем самым понять его формат. Команда info даст тебе представление о системе вцелом и в частности
о том, что все perm файлы - одинаково устроены. Это очень важный момент. Далее, тебе нужно найти файл, в котором
хранятся все учетные записи пользователей. Он находится по пути: usr/bin/users. Местный файл прав не даст тебе его
просмотреть. Но вот тут-то есть одна уязвимость. Если посмотреть на доступ к файлу perm с помощью команды gfp (get file
permissions), то можно увидеть, что гость может модифицировать файл прав - perm. Помня формат открытого perm, ты можешь
посчитать смещение по которому тебе нужно выставить 1 для этого же файла (perm), чтобы просмотреть его. Это можно сделать
с помощью команды: uf perm 151 111. После чего, файл perm будет доступен для чтения. Таким же образом нужно обновить
блок данных касающийся файла users. uf perm 192 111. После этого файла users будет доступен для чтения. Из него, ты
сможешь получить MD5 пароля для пользователя jack. Он и есть администратор системы. Для того чтобы подобрать пароль,
нужно будет снова написать небольшую подбиралку паролей. Должен получиться "Love is".
Пля попадания в следующий уровень вводим: connect jack "Love is". Вуаля :)

Уроверь 4.
Итак мы на орбитальном спутнике. После видео послания от дока, ты понимаешь, что это огромное хранилище баз данных о
всех людях. Базы данных - это файла на твоем, а так же на других спутниках. Твоя задача удалить все файлы со своего и
всех удаленных спутников тоже. Для этого нужно с ними соединиться, синхронизироваться, удалить файлы и снова
синхронизироваться. Так мы удалим все файлы на всех спутникамх. Итак по порядку:
Соединение.
Соединение работает только через землю. По-этому её нужно найти :) Самый простой способ найти землю - двигаться
влево с помощью команды left XXX. Где XXX - это дистанция. На расстоянии примерно в 700 точек, на наткнемся на нашу
планету :). После того, как она попадает в зону видимости спутники становятся доступными и можно воспользоваться
командой connect s1 s2 s3 s4 s5. Так мы соединимся со всеми спутниками сразу.
Синхронизация.
Синхронизация доступна только пока есть соединение со спутниками. Набираем sync a1 a2 a3 a4 a5 и смотрим сколько файлов
появилось у нас на локали: list. Должно появиться 13 файлов.
Удаление.
Удаляем все файлы на локали: remove facebook...google. Результат команды list должен дать: "There are no files on the
local satellite."
Теперь синхронизируемся еще раз, чтобы удалить файлы с удаленных спутников: sync a1 a2 a3 a4 a5. Всё!
